# Tour Admin Autofill ‚Äî Chrome Extension Context

## What this is

A Chrome extension that reads tour data from a Google Sheet and Google Docs, then autofills an Angular admin panel form at `/admin/product/add`. Staff previously copied data manually ‚Äî this automates it. The human reviews the filled form and clicks Save.

Access is gated behind a license key verified against a Cloudflare Worker.

## License system

### Flow

1. Popup opens ‚Üí `LicenseManager.checkLicense()` reads `lic_v1` from `chrome.storage.local`
2. If valid HMAC ‚Üí hide `panelLicense`, show app
3. If invalid/missing ‚Üí show `panelLicense` gate
4. User enters key ‚Üí `LicenseManager.activateLicense(key)` ‚Üí POST to Cloudflare Worker ‚Üí store `{key, verifiedAt, sig}`

### Background re-verification

- `background.js` schedules a Chrome alarm every 2 days
- On alarm: reads stored key, POSTs to worker, updates `verifiedAt + sig` or clears storage
- Network errors ‚Üí reschedule in 1 hour (offline-friendly grace period)

### Storage key: `lic_v1`

```js
{
  key: "LICENSE_KEY_STRING",
  verifiedAt: 1234567890000,  // ms timestamp
  sig: "deadbeef..."          // HMAC-SHA256(key + BIND_SECRET, verifiedAt)
}
```

### Cloudflare Worker

`POST https://upload-assistant.fahadmapari09.workers.dev/verify`
- Body: `{ licenseKey: "string" }`
- Response: `{ valid: true | false }`
- CORS locked to `chrome-extension://lfbleghapiiefannabbdccgenaniobgj`
- Valid keys hardcoded in worker's `VALID_KEYS` Set (see `worker.js` in parent folder)

## Google Sheets integration

- Auth: OAuth via `chrome.identity.getAuthToken()` ‚Äî no API key
- API: Sheets API v4
- Three separate calls per load:
  1. `values/{range}` ‚Äî all cell values
  2. `?includeGridData=true` ‚Äî hyperlink extraction from title column
  3. `spreadsheets/{id}` ‚Äî sheet title (display only)
- Column mapping: user configures explicit letter (A, B, AA, etc.) in settings ‚Äî NOT header-based auto-detection
- Cache: full tour array stored in `tourExtCache` in `chrome.storage.local`; replaced on manual refresh

## Google Docs integration

- Auth: OAuth via `chrome.identity.getAuthToken()` ‚Äî same token as Sheets
- API: Docs API v1 ‚Äî `GET /v1/documents/{documentId}`
- Triggered in `goToFillPanel()` when selected tour has a `docUrl`
- Parser: `parseParisDoc(docJson)` in `parisDocParser.js`

### Parser logic (`parisDocParser.js`)

- Splits document by `HEADING_1` boundaries ‚Üí one section per tour title
- Finds the correct section via fuzzy title matching (strips whitespace/punctuation)
- Extracts:
  - **List fields**: inclusions, exclusions, additionalInfo, youWillLearn, youWillSee
  - **Inline fields**: meetingPoint, duration, endLocation, coordinates (lat/lng pair)
  - **Block text**: description (accumulated until next section header)
- Handles soft line breaks (separate `textRun` elements within same paragraph)
- Skips noise: review flags (`CC OK`, `IS OK`, `SS note`), platform distribution mentions
- Switzerland variant: maps "Highlights" section header ‚Üí `youWillSee`

### Doc fields returned

```js
{
  description, inclusions, exclusions, additionalInfo,
  youWillLearn, youWillSee, meetingPoint, duration,
  endLocation, latitude, longitude
}
```

## Field filling techniques

### Native inputs & textareas

Angular intercepts `el.value = x`. Must use property descriptor:

```js
const setter = Object.getOwnPropertyDescriptor(
  HTMLInputElement.prototype, "value"
).set;
setter.call(el, value);
el.dispatchEvent(new Event("input", { bubbles: true }));
el.dispatchEvent(new Event("change", { bubbles: true }));
```

Same pattern for textareas using `HTMLTextAreaElement.prototype`.

### ng-select dropdowns

Must be filled sequentially (one at a time) ‚Äî they share a single dropdown overlay. Two strategies attempted in order:

1. **Direct component API**: `ng.getComponent(ngSelectEl).select(item)` ‚Äî fast, no DOM clicks
2. **DOM fallback**: `container.click()` ‚Üí type into search input ‚Üí wait ‚Üí click matching `.ng-option`

Cascading dropdowns (`serviceType ‚Üí activityType ‚Üí subType`, `country ‚Üí city`) require 900ms sleeps between each to allow Angular to re-render dependent options.

Multi-select: loop and call `comp.select()` for each value.

### Quill rich text editor

```js
editor.focus();
editor.innerHTML = `<p>${value.replace(/\n/g, "</p><p>")}</p>`;
editor.dispatchEvent(new Event("input", { bubbles: true }));
editor.dispatchEvent(new KeyboardEvent("keyup", { bubbles: true }));
```

### ngbdatepicker

Opens the calendar popup, navigates to the target year/month via dropdowns, clicks the matching day cell. Currently implemented but may need manual review ‚Äî adjacent-month cells are skipped.

  ‚Üí update checklist: green / red per field

## Virtual list rendering

- Row height: **64px** (fixed, required for position calculations)
- Container height: 360px visible area
- Overscan: 4 rows above and below viewport
- Only visible + overscan rows are in the DOM ‚Äî smooth performance with 100+ tours

## OAuth token management

- `getOAuthToken()` called lazily per API request
- If response is 401 or 403 ‚Üí `chrome.identity.removeCachedAuthToken()` ‚Üí re-auth
- Prevents stale or wrong-scope tokens silently failing

## Column letter ‚Üí index conversion

```
colLetterToIndex("A")  ‚Üí 0
colLetterToIndex("Z")  ‚Üí 25
colLetterToIndex("AA") ‚Üí 26
colLetterToIndex("AK") ‚Üí 36
```

Formula: iterate chars, `index = index * 26 + (charCode - 64)`, then subtract 1 for 0-based.

## Known limitations

1. **Date fields** ‚Äî ngbdatepicker fill is best-effort; adjacent-month dates are skipped. Recommend manual review of start/end dates after autofill.
2. **Doc parser tuned to Paris/Switzerland doc format** ‚Äî other doc structures may not parse correctly without updating section header patterns in `parisDocParser.js`.
3. **ng-select multi-selects** (guide languages, tags) ‚Äî currently set to single value from sheet/doc. Looping for comma-separated multi-values is possible but not yet implemented.
4. **Admin URL check** ‚Äî extension only injects into tabs at `/admin/product/add`. Will silently fail or warn on other paths.
5. **Images section** ‚Äî intentionally skipped, manual only.
